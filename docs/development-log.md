#### Spring Boot プロジェクトの実装準備

詳細設計書で整理した構成をもとに、Spring Boot アプリケーションとして実装を進めるための土台を整備した。

#### User Entity の作成

ユーザー情報を管理する `User` Entity を作成した。

以下の項目を保持する構成とした。

- ユーザーID
- 名前
- メールアドレス
- パスワード
- 作成日時
- 更新日時

メールアドレスはユーザーを一意に識別する項目として扱うため、DB上でユニーク制約を設定した。

また、作成日時と更新日時は `@PrePersist`、`@PreUpdate` を利用して自動的に設定・更新されるようにした。

#### Task Entity の作成

タスク情報を管理する `Task` Entity を作成した。

以下の項目を保持する構成とした。

- タスクID
- ユーザー
- タイトル
- 内容
- 期限日
- 優先度
- ステータス
- 作成日時
- 更新日時

タスクはユーザーに紐づく情報であるため、`User` と `Task` を 1対多 の関連として定義した。

#### UserRepository の作成

`UserRepository.java` を作成し、`JpaRepository<User, Long>` を継承してユーザー情報へアクセスできるようにした。

追加した検索メソッドは以下の通り。

- `findByEmail(String email)`
- `existsByEmail(String email)`

`findByEmail` はログイン処理などでメールアドレスからユーザーを取得するために使用する想定とした。

`existsByEmail` はユーザー登録時に、同じメールアドレスが既に登録されていないか確認するために使用する想定とした。

戻り値に `Optional<User>` を利用することで、ユーザーが存在しない場合の処理を呼び出し側で明確に扱えるようにした。

#### TaskRepository の作成

`TaskRepository` を作成し、ユーザー単位でタスクを取得できるようにした。

以下の条件で検索できるメソッドを用意した。

- ユーザーID
- ステータス
- 優先度
- 期限日
- タイトルのキーワード

タスク一覧画面や検索機能で必要になる条件を、Repository のメソッドとして先に整理した。

### 学び・気付き

Spring Data JPA では、`JpaRepository` を継承するだけで基本的なCRUD操作を利用できるため、Repository では業務上必要な検索条件を中心に定義すればよいことが分かった。

また、`findByEmail` や `existsByEmail` のようにメソッド名からクエリが生成されるため、単純な検索であれば SQL を直接書かずに実装できる点が便利だと感じた。

Entity では、テーブルの項目だけでなく、制約、関連、初期値、日時更新のタイミングも意識して定義する必要があると分かった。

#### Service 層の作成

業務ロジックを担当する Service 層として、`UserService` と `TaskService` を作成した。

`UserService` では、ユーザー登録、メールアドレス重複チェック、メールアドレス検索、ログイン認証を実装した。

`TaskService` では、タスク一覧取得、詳細取得、登録、更新、削除、ステータス更新、検索を実装した。

タスク操作では、対象タスクがログイン中のユーザーに紐づくものかを確認し、他ユーザーのタスクを操作できないようにした。

#### Controller 層の作成

画面および API からのリクエストを受け取る Controller を作成した。

作成した Controller は以下の通り。

- `AuthController`
- `TaskController`
- `PageController`

`AuthController` では、ユーザー登録、ログイン、ログアウト用の API を実装した。

`TaskController` では、タスクの一覧取得、詳細取得、登録、更新、削除、ステータス更新用の API を実装した。

`PageController` では、ログイン画面、ユーザー登録画面、タスク一覧画面、タスク登録画面、タスク編集画面、タスク詳細画面への遷移を実装した。
