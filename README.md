# 個人向けタスク管理システム

## プロジェクト概要

個人またはチームのタスクを管理するための Web アプリケーションです。

タスクの登録、一覧表示、編集、削除、検索を行い、日々の作業状況を整理できることを目的とします。

## システムの目的

- 個人またはチームのタスクを一元管理する
- タスクの期限、優先度、ステータスを見える化する
- 必要なタスクを検索しやすくする
- Docker を使って開発環境を簡単に起動できるようにする

## 利用者

| 利用者 | 内容 |
| --- | --- |
| 一般ユーザー | 自分のタスクを登録、確認、編集、削除する |
| 管理者 | ユーザーやタスクの管理を行う |

## 業務課題

- タスクが個人のメモやチャットに散らばり、状況を把握しづらい
- 期限や優先度が見えにくく、対応漏れが発生しやすい
- 完了済み、対応中、未着手などの状態管理が曖昧になりやすい
- 必要なタスクを後から探すのに時間がかかる

## 機能要件

| No | 機能 | 内容 |
| --- | --- | --- |

## 単体テスト状況

JUnit による単体テストは、現在の実装範囲に対して実施済みです。

確認済みの主な内容は以下です。

- ログイン成功、パスワード不一致

- ステータス変更
- タイトル、ステータス、優先度などの検索
- 所有ユーザー以外のタスクを取得しない確認
- Repository、Service、Controller の正常系・異常系

直近のテスト結果は以下です。

- テスト件数: 36件

- エラー: 0件
- スキップ: 0件

ただし、`docs/単体テスト仕様書.xlsx` と完全一致している状態ではありません。

仕様書にはあるが、現在の実装またはJUnitテストでは未対応の主な項目は以下です。

- JWT生成処理
- `PasswordEncoder` によるパスワード照合
- PDF出力機能
- `TaskPdfService`
- `TaskService` の一部異常系

現在のアプリケーションは仮トークン方式で認証しているため、JWTや `PasswordEncoder` の項目は今後の実装方針に合わせて、実装対象にするか仕様書側を見直す必要があります。
